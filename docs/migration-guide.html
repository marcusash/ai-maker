> MIGRATION GUIDE
Upgrading from the CLI setup to the GitHub Copilot App
> YOUR DATA IS SAFE
Your original files are never deleted. Migration copies your vault and settings to the new location , your originals stay exactly where they are until you decide to remove them yourself.
We copy. We never move. We never delete.
> WHAT YOU HAVE NOW
On your PC
C:\AIMaker\ , AI maker workspace, vault, settings
C:\AIWorkbench\ , only if you ran a custom setup (most users don't have this)
Windows Terminal profiles (blue and red tabs)
Desktop shortcut → launch.ps1
GitHub Copilot CLI installed
On GitHub (Red Pill users)
Private pc-setup repo with your machine config and vault backup
> WHAT CHANGES
What goes away
Dual Terminal tabs (replaced by the Copilot App)
Desktop shortcut (pin the App to taskbar instead)
C:\AIMaker\ folder , consolidated into one project folder at C:\GitHub\ai-workspace
Old skills in .github/skills/ , superseded by App-global skills in ~/.copilot/skills/
What stays the same
Your vault , all your saved notes and context
Your persona , copilot-instructions.md carries over
Your GitHub repos , pc-setup is never touched
Copilot CLI , still available for terminal-first work
All your saved work , nothing is lost
> WHAT YOU GAIN
The App does everything the CLI does. It's a better interface for the same work , plus capabilities you didn't have before.
Visual chat interface
Talk to your AI in a real window. Formatted responses, inline code, rendered canvases. No more reading walls of terminal text.
Multiple parallel sessions
Run research in one session, code review in another, writing in a third. All at the same time. No context switching, no losing your place.
Inter-agent messaging
Sessions can talk to each other. One agent can hand off work to another mid-task. This is a new capability , the CLI had no equivalent.
Project awareness
The AI can see and edit your files directly. Open a folder as a project and your agent reads, writes, and reasons about your actual work.
Skills always available
All 11 (or 22) skills are active in every session, across every project. No more cd-ing to the right workspace folder first.
Built-in terminal
Full PowerShell access in a side panel. Everything the CLI can do, right next to your chat.
Cross-machine setup (Red Pill)
Your skills and vault sync via Git. Set up a new machine in minutes, authenticate with GitHub and the installer restores everything from your private repo.

> HOW TO MIGRATE
1
Preview what will change (optional but recommended)
Open PowerShell and run:
$f="$env:TEMP\migrate.ps1"; irm https://github.com/marcusash/ai-maker/releases/download/v3.0.5/migrate.ps1 -OutFile $f; & $f -WhatIf
This downloads the script and runs it in preview mode , shows exactly what will copy and where, without touching anything. Review before committing.
2
Run the migration script
Or choose option 3 (Migrate) when you run install.bat:
irm https://github.com/marcusash/ai-maker/releases/download/v3.0.5/migrate.ps1 | iex
The script detects your existing setup automatically. It shows you a summary of what will change and asks for confirmation before copying anything.
3
Confirm the migration
The script prints exactly what it will copy and where. Type Y to proceed. Your vault copies to C:\GitHub\ai-workspace\vault\, your persona copies to C:\GitHub\ai-workspace\.github\copilot-instructions.md. Your original C:\AIMaker folder remains untouched. C:\AIWorkbench is not required , if you don't have it, the script skips it automatically.
4
Open the Copilot App
Launch the GitHub Copilot App and add C:\GitHub\ai-workspace as a project. Open a new session. Your agents bootstrap themselves with your migrated vault and persona.
5
Verify everything looks right
Ask your agent: "What do you remember about me?" and "What's in my vault?" Once you've confirmed everything is there, you can remove C:\AIMaker and C:\AIWorkbench at your own pace. There's no rush , they're just taking up disk space.
> QUESTIONS
What if I have a custom copilot-instructions.md?
The script checks if you've modified it. If you have, your version wins , it copies as-is. If you're still on the stock version, it upgrades to v3.
What about my GitHub repo?
Your existing pc-setup repo is never touched. The migration creates a new private ai-workspace repo (Red Pill users) , it does not rename, fork, or modify pc-setup.
Can I go back?
Yes. Your original folders are untouched. The Copilot CLI is still installed. You can keep using the terminal workflow alongside the App , there's no forced cutover.
Something went wrong. What do I do?
The migration script logs every operation. If something fails, the log tells you exactly what happened and what to fix. Your original files are always intact because we copy, not move.
