>>> SCENARIO A: HOW IT WORKS <<<
> THE BIG CHANGE
Before, migration was a separate code path that did its own install. It got out of sync with the regular installer, missed pieces like WorkIQ and Agency, and left users in a half-finished state.
Now, migration is a thin wrapper around the regular install. It backs up your old content, runs the full Blue or Red Pill installer (the same one a fresh user runs), then restores your content into the new workspace. You end up in exactly the same known-good state as a fresh install, just with your data preserved.
1. Detect your old install (Blue or Red flavor)
v
2. Copy your vault + persona to a safe backup location
v
3. Run the full installer (App, skills, WorkIQ, Agency, workspace, MCP wiring)
v
4. Restore your vault + persona into the new workspace
v
5. Done. Same end state as a fresh install. Your old folder is still there, untouched.
What "known-good state" means
Copilot App installed, all 11 (or 22) skills loaded, WorkIQ authed, Agency MCP servers wired up, workspace folder created with your persona and vault content, GitHub repo connected (Red Pill only).

>>> READY TO MIGRATE? <<<
> HOW TO MIGRATE
1
Preview what will happen (optional but recommended)
Open PowerShell and run:
$f="$env:TEMP\migrate.ps1"; irm https://github.com/marcusash/ai-maker/releases/download/v3.0.6/migrate.ps1 -OutFile $f; & $f -WhatIf
Downloads the script and runs it in preview mode. Shows you which flavor it detected (Blue or Red), what will be backed up, and which installer it will hand off to. Nothing changes on your machine.
2
Run the migration
When you're ready:
irm https://github.com/marcusash/ai-maker/releases/download/v3.0.6/migrate.ps1 | iex
The script detects your existing setup, identifies your flavor, and shows a summary. Type Y to proceed.
3
Backup phase
Your vault and persona are copied to a safe location at %TEMP%\ai-maker-migration-backup\. Your original folder is not touched.
4
Install phase
The migration script hands off to the regular installer (install-blue.ps1 or install-red.ps1, depending on your flavor). This installs the GitHub Copilot App, all skills, WorkIQ, Agency MCP servers, the workspace folder, and (for Red Pill) creates your ai-workspace GitHub repo. Same exact install a fresh user gets.
5
Restore phase
Your vault and persona are copied from the backup into the new workspace at C:\GitHub\ai-workspace\. If your copilot-instructions.md was customized, your version wins. If it was stock, the new v3 version is used.
6
Verify and clean up at your own pace
Open the Copilot App. Add C:\GitHub\ai-workspace as a project. Ask your agent: "What do you remember about me?" Once you've confirmed everything is there, you can delete C:\AIMaker whenever you want. No rush. It's just taking up disk space.
>>> HOW WE TESTED THIS <<<
> TEST RIGOR
We don't ship a migration without testing it on a real-looking old install. Two helper scripts make this repeatable:
populate-old-state.bat — simulate a pre-migration user
Creates C:\AIMaker\ (and C:\AIWorkbench\ for Red flavor) with realistic content: vault entries, persona file, sample skills, recognizable marker strings so we can verify nothing was lost.
Flags: -Flavor blue or -Flavor red.
reset.bat — full wipe
Removes everything: App, skills, workspace, legacy folders, registry entries. Gets you back to a clean slate so you can populate again and re-test.
The test loop
1. reset.bat — clean slate
2. populate-old-state.bat -Flavor blue — fake old install
3. migrate.ps1 — run the migration
4. Check the new workspace for the marker strings — verify nothing was lost
5. Repeat with -Flavor red
> QUESTIONS
What if I have a custom copilot-instructions.md?
The script checks if you've modified it. If you have, your version wins, it copies as-is. If you're still on the stock version, it upgrades to v3.
What if the install phase fails halfway through?
Your backup is still in %TEMP%\ai-maker-migration-backup\. Your old folder is still intact. Fix whatever the installer complained about (usually a winget issue) and re-run migrate.ps1, it picks up where it left off.
What about my pc-setup GitHub repo?
Never touched. The Red Pill installer creates a new ai-workspace repo, separate from pc-setup.
Can I go back?
Yes. Your old folder is untouched. The Copilot CLI is still installed. You can keep using the terminal workflow alongside the App.
Why isn't there a "mark legacy" step on my old folder?
The new App lives in a completely different folder (C:\GitHub\ai-workspace), so there's no name collision. Stamping your old folder with a "legacy" tag adds nothing and risks surprising you. We leave it exactly as we found it.
